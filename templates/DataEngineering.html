Climate Vulnerability Data Engineering

Data preparation, cleaning, pipeline transformations, and feature engineering applied to historical disaster records for machine learning readiness.

Phase Introduction

Purpose of the Phase: Data Engineering operationalizes the project by preparing raw repositories for machine learning workflows. This stage transforms sparse environmental incident registers into structured, statistically consistent inputs optimized for predictive territorial classification.

Applied Preprocessing & Data Cleaning

01
Deduplication: Identifying and purging cross-reported emergency entries to safeguard sample independence.
02
Geographic Standardization: Mapping municipal strings to official national code indexes (DIVIPOLA) to correct regional name variants.
03
Lexical Normalization: Resolving categorical text discrepancies stemming from legacy system encodings and miscellaneous input formats.
04
Imputation & Missingness: Assessing sparsity within geographic metrics and timestamps, discarding unresolvable records.
05
Temporal Parsing: Standardizing string-based date configurations into structured ISO datetime formats for seasonal analysis.

Feature Engineering & Transformation

Spatial Aggregation
Grouping specific incident arrays into municipal boundaries to generate regional baselines.
Risk Indicators
Deriving cumulative risk exposure indexes based on frequency weights and impact severities.
Event Categorization
Mapping irregular event descriptions into high-level meteorological hazard domains.
Temporal Discretization
Extracting seasonal components, quarter slices, and yearly counts to isolate cyclic variations.
Density Scoring
Formulating localized density parameters tracking continuous disaster recurrence limits.
Feature Scaling
Normalizing engineered numeric arrays to prevent distance biases in clustering algorithms.
