Logistic Regression Concepts

Theoretical foundation of the Logistic Regression algorithm — its formulation, training strategy, hyperparameters, variables, and application to climate vulnerability classification in Colombian municipalities.

Algorithm Type

Supervised Classification

Strategy

Multinomial (3 Classes)

Framework

scikit-learn · lbfgs

What is Logistic Regression?

Despite its name, Logistic Regression is a classification algorithm, not a regression one. It models the probability that a given observation belongs to a particular class by applying a non-linear transformation — the sigmoid or softmax function — to a linear combination of input features.

Core idea: Find a set of weights β such that the linear combination z = β₀ + β₁x₁ + β₂x₂ + … + βₙxₙ, when passed through the sigmoid function, produces a probability value between 0 and 1 for each class.

The sigmoid σ(z) = 1 / (1 + e−z) maps any real value to (0, 1). For multiple classes the softmax generalisation is applied.

# Binary sigmoid
σ(z) = 1 / (1 + e−z)

# Multinomial softmax (k classes)
P(y=k | x) = exp(βk·x) / Σⱼ exp(βj·x)

# Final decision
ŷ = argmaxk P(y=k | x)

# Cross-entropy loss (minimised during training)
L = −Σᵢ Σₖ yᵢₖ · log( P(yᵢ=k | xᵢ) )

Hyperparameters Used

The model was instantiated with the following configuration in scikit-learn. Each parameter was chosen to match the scale and multiclass nature of the UNGRD dataset.

Parameter Value Used Explanation
solver lbfgs Limited-memory Broyden–Fletcher–Goldfarb–Shanno optimiser. Efficient for small-to-medium datasets and natively supports multinomial loss.
max_iter 1000 Maximum optimisation iterations. Increased from the default (100) to ensure convergence given 15 features and 990 samples.
C 1.0 Inverse of regularisation strength. C = 1.0 (default) applies moderate L2 regularisation, balancing bias and variance.
penalty l2 L2 (Ridge) regularisation penalises large coefficients, improving generalisation to unseen municipalities.
random_state 42 Seed for reproducibility across training runs and experiments.
multi_class auto With lbfgs, scikit-learn automatically selects the multinomial strategy, computing a joint softmax probability over all 3 classes simultaneously.
Regularisation note: The C parameter controls the trade-off between fitting the training data and keeping model weights small. Lower C → stronger regularisation → simpler model. Higher C → less regularisation → fits training data more closely but risks overfitting.

How the Model Was Trained

Training followed a standard supervised learning pipeline with careful preprocessing to ensure feature comparability across municipalities of very different sizes and disaster frequencies.

1
Data Loading & Label Consolidation
The UNGRD Scaled dataset (10 802 records) was loaded. The original 4-level label Critical was merged into High to produce exactly 3 balanced vulnerability classes: Low, Medium, High.
2
Municipality-Level Aggregation
Records were grouped by MUNICIPALITY (990 unique municipalities). Affectation columns were summed, index columns were averaged, and count columns took the maximum. The target label was assigned as the dominant class (mode) of all records for that municipality.
3
Stratified Train / Test Split
Data split into 80% training (792 municipalities) and 20% test (198 municipalities) using stratify=y to preserve class proportions in both sets.
4
Feature Scaling
StandardScaler was fit only on training data (to avoid data leakage) and applied to both train and test sets. This centres each feature to mean = 0 and standard deviation = 1, which is critical for gradient-based optimisers like lbfgs.
5
Model Fitting
LogisticRegression(solver="lbfgs", max_iter=1000) was fitted on the scaled training set. The lbfgs optimiser iteratively minimises the multinomial cross-entropy loss, updating the 15 × 3 coefficient matrix.
6
Evaluation & Export
Predictions on the test set were evaluated with accuracy, classification report (precision/recall/F1 per class), confusion matrix, and ROC-AUC (One-vs-Rest). Final predictions for all 990 municipalities were exported to CSV.

Vulnerability Classes

The model classifies each municipality into one of three ordered levels.

Low
Few recorded events, low severity index, minimal human and infrastructure impact. Municipality has strong adaptive capacity.
Medium
Moderate recurrence of climate events. Significant but manageable affectation. Requires prevention investment.
High
High severity and recurrence. Notable casualties, destroyed housing, critical sector damage. Demands prioritised intervention.

Input Variables (Features)

15 features were selected from the UNGRD Scaled dataset, grouped into four thematic categories. All were standardised before model training.

SEVERITY_INDEX
Composite score of event intensity. Strongest predictor in the model.
VULNERABILITY_INDEX
Pre-computed composite vulnerability score aggregating multiple risk dimensions.
AVG_SEVERITY_MUNICIPALITY
Mean severity across all historical events recorded in the municipality.
PEOPLE
Total number of persons affected by climate events (summed across records).
FAMILIES
Number of families displaced or affected by disasters.
DECEASED
Cumulative fatalities attributed to climate emergencies.
INJURED
Number of persons injured during recorded events.
DAMAGED HOUSES
Houses with partial structural damage — key housing impact indicator.
DESTROYED HOUSES
Houses completely destroyed; reflects severity of physical impact.
DAMAGED ROADS
Road infrastructure damage, affecting emergency response capacity.
EDUCATIONAL CENTERS
Schools affected; indicates community-level social disruption.
HECTARES
Agricultural land area affected — relevant for rural municipalities.
MUNICIPALITY_EVENT_COUNT
Total number of distinct climate events registered in the municipality.
EVENT_DIVERSITY
Number of different event types recorded (floods, landslides, drought, etc.).
RAINY_SEASON
Proportion of events occurring during Colombia's rainy seasons (bimodal).

Advantages & Limitations

Logistic Regression offers strong interpretability and solid baseline performance, but has inherent constraints when applied to complex, non-linear environmental data.

Advantages

Interpretable coefficients: Each feature weight directly indicates direction and relative magnitude of its contribution to class probability.
Probabilistic output: Returns calibrated probabilities per class, enabling uncertainty quantification for each municipality's classification.
Computationally efficient: Fast training and inference — suitable for rapid re-training as new UNGRD records become available.
Robust baseline: Performs well when classes are approximately linearly separable, as observed with the vulnerability indices in this dataset (93.4% accuracy).
Regularisation built-in: L2 penalty prevents overfitting on the 990-municipality dataset without requiring external techniques.

Limitations

Linear decision boundary: Assumes classes are linearly separable in feature space. Complex non-linear climate interactions may not be captured.
Medium class overlap: The Medium vulnerability class showed lower recall (74%), reflecting its transitional nature between Low and High.
Feature independence assumption: Correlated features (e.g. PEOPLE and FAMILIES) can distort coefficient magnitudes without prior decorrelation.
Static aggregation: Municipality-level aggregation loses temporal dynamics — seasonal vulnerability patterns and event trends are not modelled.
No spatial context: The model treats municipalities as independent observations, ignoring geographic proximity effects and regional spillovers.

Application to Climate Vulnerability in Colombia

Colombia's bimodal rainfall regime, Andean topography, and Pacific/Caribbean coastlines create a uniquely complex climate risk landscape. The UNGRD dataset captures this diversity across more than 1 000 municipalities spanning 32 departments.

🗺️
Territorial Risk Prioritisation

By classifying municipalities into Low / Medium / High vulnerability, UNGRD and territorial planning entities can allocate emergency prevention budgets proportionally to actual risk exposure, reducing response times and improving pre-disaster preparedness.

🌧️
Rainy Season Readiness

The RAINY_SEASON feature captures Colombia's bimodal precipitation cycle (April–June, October–December). Municipalities showing high event concentration during these windows receive elevated vulnerability scores, enabling seasonal pre-positioning of humanitarian resources.

🏘️
Housing & Infrastructure Vulnerability

Variables like DAMAGED HOUSES and DESTROYED HOUSES directly reflect housing stock fragility — a key indicator for municipalities with precarious informal settlements in flood-prone or landslide-risk zones, particularly in the Pacific and Andean regions.

📡
Scalable Early Warning Integration

The pre_resultado() prediction function can receive real-time municipal indicators and return a vulnerability class + probability triplet, making it suitable for integration with national early warning dashboards or IDEAM climate monitoring systems.

📊
Decision Support for Public Policy

The model's interpretable coefficients allow environmental authorities to understand which risk factors drive high vulnerability in specific regions — for instance, whether it is event frequency (Chocó) or infrastructure damage (Cauca valleys) — enabling targeted policy instruments rather than one-size-fits-all emergency plans.

Theoretical Summary: Logistic Regression provides a transparent, probabilistic, and computationally efficient framework for municipal vulnerability classification. The multinomial softmax formulation, combined with L2 regularisation and standardised UNGRD features, achieved 93.4% accuracy — validating the algorithm's suitability as a core modelling choice within this CRISP-ML pipeline. Future phases may explore ensemble methods (Random Forest, XGBoost) or spatial models (GWR) to address the current model's linearity and spatial independence constraints.
