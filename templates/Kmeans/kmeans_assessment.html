Core Metrics

Accuracy
Global
Accuracy = (TP + TN) / (TP + TN + FP + FN)
Measures the proportion of correctly classified instances over the total. Simple and intuitive, but unreliable with imbalanced datasets — a model predicting all negatives can still score high accuracy.
0
1
Precision
Positive
Precision = TP / (TP + FP)
Of all instances predicted as positive, what fraction actually were positive? High precision means fewer false alarms. Critical when the cost of a False Positive is high (e.g. flagging a safe municipality as high-risk).
0
1
Recall
Sensitivity
Recall = TP / (TP + FN)
Of all actual positives, what fraction did the model detect? High recall minimizes False Negatives. Critical when missing a real risk is costly — e.g. not detecting a high-vulnerability municipality.
0
1
F1-Score
Harmonic Mean
F1 = 2 × (Precision × Recall) / (Precision + Recall)
The harmonic mean of Precision and Recall. Balances both metrics into a single score. Preferred over accuracy when classes are imbalanced. F1 = 1 is perfect, F1 = 0 is worst.

Error Metrics

MAE
Mean Absolute Error
MAE = (1/n) × Σ |yᵢ − ŷᵢ|
Average absolute difference between predicted and actual values. Easy to interpret (same units as the target). Not sensitive to outliers — all errors are weighted equally regardless of magnitude.
MSE
Mean Squared Error
MSE = (1/n) × Σ (yᵢ − ŷᵢ)²
Squares the errors before averaging — penalizes large errors more heavily. Differentiable, making it ideal for gradient-based optimization. Units are squared, making interpretation harder than MAE.
RMSE
Root Mean Squared Error
RMSE = √[ (1/n) × Σ (yᵢ − ŷᵢ)² ]
Square root of MSE — restores original units for interpretability while still penalizing large errors. Most commonly reported metric in regression tasks alongside R².
R²
Coefficient of Determination
R² = 1 − (SS_res / SS_tot)
Proportion of variance in the target explained by the model. R² = 1 means perfect fit; R² = 0 means the model is no better than predicting the mean; negative R² means worse than the mean baseline.
