Model Explanation

What is K-Means?

K-Means is an unsupervised clustering algorithm that partitions a dataset into K groups (clusters), where each observation belongs to the cluster whose centroid is closest in Euclidean space. It iteratively refines cluster boundaries until convergence, making it ideal for discovering hidden vulnerability patterns across municipalities.

Objective Function

Minimize inertia (within-cluster sum of squares)
J = Σ Σ ‖xᵢ − μₖ‖²

where xᵢ = data point, μₖ = centroid of cluster k
Advantages
Simple and computationally efficient
Scales well with large datasets
Works without labeled training data
Easy to interpret cluster centroids
K-Means++ improves initialization quality
Limitations
Requires predefined number of clusters K
Sensitive to outliers and scale
Assumes spherical, equal-size clusters
May converge to local optima
Not suitable for categorical data directly

Training Process

01
Data Preparation
Load and normalize the UNGRD dataset. Apply StandardScaler so all features contribute equally regardless of magnitude.
02
Train/Test Split
Divide data 80% / 20% using random_state=42 for reproducibility. The model trains only on the training split.
03
K-Means++ Init
Centroids are seeded using K-Means++ strategy, spreading initial points to improve convergence speed and quality.
04
Iterative Assignment
Each data point is assigned to its nearest centroid. Centroids are then recalculated as the mean of assigned points.
05
Convergence
Repeat assignment and update steps until centroids stop moving (max 300 iterations, 10 random initializations).
06
Silhouette Validation
Evaluate cluster quality using Silhouette Score on training data. Score ranges from -1 (poor) to +1 (excellent).

Hyperparameters

Parameter Value Purpose
n_clusters 3 (default) Number of vulnerability groups: Low, Medium, High
init k-means++ Smart centroid initialization to avoid bad local optima
max_iter 300 Maximum iterations before forced convergence
n_init 10 Number of independent runs; best result is kept
random_state 42 Seed for reproducibility across executions
